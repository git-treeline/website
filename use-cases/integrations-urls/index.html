Use cases / Integrations

Redirects and webhooks care about the URL, not your branch name

OAuth providers, payment webhooks, and mobile deep links are configured with concrete hostnames—or sometimes a single fixed localhost port. Parallel worktrees fight that model because each branch gets its own allocated port. You choose the layer that matches the integration’s constraints.

Pick a tool

Need Command
Stable hostname per branch on your machine gtl serve
Provider only allows localhost:3000 gtl proxy
Public HTTPS on your domain, same route shape as local gtl tunnel
Send someone a temporary link; tear down when done; URL not guessable from branch gtl share (or --tailscale for tailnet-only)

gtl tunnel is for ongoing, routable access—think webhooks and OAuth redirect URIs you register once against your domain. gtl share is for a disposable session: token-gated in the default Cloudflare mode, cleaned up on Ctrl+C. Frameworks still need host allowlists; Treeline prints the exact change for Rails, Vite, and Django when you run tunnel.
