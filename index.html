Everything a worktree needs to run

Each capability is opt-in. Declare what your project needs in .treeline.yml.

.treeline.yml 
# Ports
port: auto
extra_ports: 1

# Database
database:
  engine: postgresql
  name: myapp_dev

# Environment
env_file: .env.local
copy_files:
  - config/master.key

# Lifecycle
commands:
  setup: bin/setup
  start: bin/dev

# Editor
editor: auto

Zero footprint

  • No runtime dependency. CLI tool, not a library. Writes a .env file and walks away.
  • No lock-in. .treeline.yml is declarative config. Non-users just ignore it.
  • No magic. No git hooks, no daemons, no implicit behavior.
  • Opt-in per developer. Commit the config, let the team decide.

Built for AI agents

  • Lifecycle hooks. gtl setup . on creation, gtl release . --drop-db on teardown.
  • Machine output. --json on read commands. Structured data, no parsing.
  • Agent context. gtl init writes AGENTS.md for Cursor, Claude Code, and Codex.

Questions

What does git-treeline do?

Git Treeline is a CLI that sets up isolated development environments per git worktree — unique ports, cloned databases, Redis namespaces, env files, and setup commands. One command turns a bare checkout into a bootable copy of your app.

Do I need to use all the features?

No. Every capability is opt-in. If your app only needs port allocation and an env file, that's a three-line .treeline.yml.

How do I review a GitHub PR with git-treeline?

Run gtl review <PR-number>. Treeline uses the gh CLI to fetch the PR, creates a worktree for it, allocates isolated resources, and runs setup. Add --start to boot the app immediately. When you're done, gtl release --drop-db cleans everything up.

Does it work with AI coding agents?

Yes. Add gtl setup . on worktree creation and gtl release . --drop-db on teardown. --json output for programmatic access, and gtl init generates agent context files for Cursor, Claude Code, and Codex.

Does git-treeline add runtime dependencies to my app?

No. It's a standalone CLI binary that writes an .env file at setup time. Nothing ships in your bundle, Docker image, or deploy.

What if some team members don't use git-treeline?

Nothing changes for them. .treeline.yml is a config file they can ignore. The app boots on default ports with the default database.
