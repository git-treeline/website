When to use what

gtl serve

Every worktree at {branch}.localhost with HTTPS. Always-on background service, zero config per branch.

gtl proxy

Fixed port forwarding to a dynamic worktree port. For OAuth callbacks, webhooks, and anything that expects a known URL.

gtl tunnel

Share a running branch publicly via Cloudflare. Branch-named subdomains on your own domain, or a quick random URL.

gtl serve

Maps every worktree to {project}-{branch}.localhost with trusted HTTPS certificates. One-time install, then it runs in the background forever.

$ gtl serve install

All set! Router is running in the background.

https://{project}-{branch}.localhost

Full reference: install, status, how it works, uninstall →

gtl proxy

Binds a fixed port and forwards to the current worktree. OAuth, Stripe, webhooks all point to :3000 regardless of which branch is running.

$ gtl proxy 3000

Proxying :3000 → :3010

Full reference: TLS, auto-detection, protocol support →

gtl tunnel

Expose a running branch to the public internet via Cloudflare. No account needed for a quick random URL. After one-time setup, branch names become subdomains on your domain.

Quick mode

$ gtl tunnel

https://random-slug.trycloudflare.com

Named mode (your domain)

$ gtl tunnel

https://myapp-feature-auth.myteam.dev

Full reference: setup walkthrough, named tunnels, route keys →

URLs are derived from your project name and branch using a shared route key convention. Local and public URLs follow the same naming.
